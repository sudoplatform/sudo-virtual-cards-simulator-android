Simulate a refund transaction.

Simulating a refund will generate a refund transaction. Simulating a refund does not modify any existing records, and instead generates a new refund transaction.

Refunds can only be performed against a transaction that has already been previously debited. Debits can also be partially refunded, which will generate a new refund transaction record of the partially refunded amount.

Refunds cannot exceed the total amount of a debit, otherwise an error will be returned and no operation will be performed.

Return

The details of the refund

Parameters

request

The data needed to create the refund
